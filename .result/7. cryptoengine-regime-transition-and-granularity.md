### 판단 기준 적용
```
채택 기준: 6/8개 레짐의 OOS Sharpe > 4개 + 0.3 이상, AND consistency 유지

4개 → 6개: Sharpe 1.100 → 1.042 (delta = -0.058) → 기준 미달
4개 → 8개: Sharpe 1.100 → 1.046 (delta = -0.054) → 기준 미달
```

### 결론: **4개 레짐 유지**

```
4개 유지 권장 (단순함 우선)
더 세밀한 레짐 분류가 OOS Sharpe를 기준(1.100) + 0.3 이상 개선하지 못함.
```

**세분화 실패 원인:**
1. **레짐 전환 빈도 증가**: 4개(85.5/1000봉) → 6개(114.5) → 8개(136.5)
   → 세분화할수록 더 자주 바뀌고, Method B에서도 자본 재배분 지연 길어짐
2. **`volatile` 레짐 고착**: 4/6/8 모두 volatile이 3.1% 고정 → volatile 분류 자체가 부족
3. **strong_down 통찰**: 6개 레짐에서 strong_down이 20.7% — BTC 하락 기간이 생각보다 길었음
4. **파라미터 과적합 위험**: 파라미터 수 12→18→24로 증가하나 성과는 오히려 하락
