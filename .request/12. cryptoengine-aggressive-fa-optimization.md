### B-2. FA 자본 비율이 수익에 미치는 영향 (이론)

```
FA 자본 50%: 자본의 50%로 펀딩비 수취 → 연 8.07% × 0.50 = 포트폴리오 연 4.04%
FA 자본 70%: 자본의 70%로 펀딩비 수취 → 연 8.07% × 0.70 = 포트폴리오 연 5.65%
FA 자본 90%: 자본의 90%로 펀딩비 수취 → 연 8.07% × 0.90 = 포트폴리오 연 7.26%

단, 이 계산은 선형 확장을 가정함.
실제로는:
1) 마진 부족으로 포지션 크기 제한 가능
2) 현금 버퍼 부족으로 마진콜 위험 증가
3) 포지션 크기 증가 시 슬리피지 증가 가능
```

## Stage C: 레버리지 한계 테스트

### 목적

FA 전략의 레버리지를 2배에서 단계적으로 올려서, **청산 없이 버틸 수 있는 최대 레버리지**를 찾는다.

### C-1. FA 레버리지의 특수성

```
일반 방향성 전략의 레버리지:
  레버리지 ↑ → 가격 반대 움직임 시 손실 ↑ → 청산 위험 ↑
  
FA (델타 뉴트럴)의 레버리지:
  현물 롱 + 선물 숏 → 가격 변동 상쇄 → 이론상 가격 리스크 없음
  
  그러면 왜 2배로 제한했는가?
  
  청산 리스크가 0이 아닌 3가지 이유:
  1) 베이시스 스프레드 급변동 — 현물과 선물 가격이 일시적으로 크게 벌어짐
  2) 편측 체결 — 한쪽 레그만 체결되어 방향성 노출
  3) 마진 부족 — 선물 포지션의 유지증거금이 가용 마진 초과
  
  이론상 10배도 가능하지만, 위 3가지 리스크 때문에 보수적으로 제한 중.
  이번 테스트에서 실제 데이터 기반으로 한계를 확인한다.
```

### C-3. 청산 시뮬레이션

```python
class LiquidationSimulator:
    """
    각 레버리지에서 6년간 청산이 발생하는지 시뮬레이션한다.
    
    Bybit 유지증거금 (BTCUSDT):
    - 포지션 ≤ $2M: 유지증거금률 0.5%
    - 포지션 $2M~$4M: 1.0%
    
    청산 조건:
    마진비율 = (계좌잔고 + 미실현손익) / 유지증거금
    마진비율 ≤ 100% → 청산
    
    FA 미실현손익 계산:
    - 현물 롱: (현재 현물가 - 진입 현물가) × 수량
    - 선물 숏: (진입 선물가 - 현재 선물가) × 수량
    - 합산 미실현손익 ≈ 베이시스 변동 × 수량 (가격 변동은 상쇄)
    """
    
    def simulate(self, leverage, fa_capital_ratio, ohlcv, funding_rates):
        equity = 10000
        fa_capital = equity * fa_capital_ratio
        
        for trade in fa_trades:
            position_notional = fa_capital * leverage
            maintenance_margin = position_notional * 0.005  # 0.5%
            
            # 매 시간 베이시스 스프레드 변동에 따른 미실현 손익 계산
            for hour in trade.holding_period:
                basis_change = get_basis_change(hour)
                unrealized_pnl = position_notional * basis_change
                
                margin_ratio = (fa_capital + unrealized_pnl) / maintenance_margin
                
                if margin_ratio <= 1.0:
                    record_liquidation(hour, margin_ratio, leverage)
                    break
                    
                # 마진콜 경고 (150% 이하)
                if margin_ratio <= 1.5:
                    record_margin_warning(hour, margin_ratio, leverage)
```

## 주의사항

1. **청산 0회가 절대 기준**: 수익이 아무리 높아도 6년 중 1회라도 청산 발생하면 해당 설정은 불가.
2. **베이시스 리스크는 가격 리스크와 다름**: FA는 가격 변동에 중립이지만 베이시스 변동에는 노출됨. 레버리지가 높을수록 베이시스 리스크가 커짐.
3. **편측 체결은 레버리지에 비례하여 위험**: 레버리지 10배에서 편측 체결 3분이면 자본의 10%+ 손실 가능.
4. **현물 재투자 MDD는 원금 손실이 아님**: 재투자 비율을 높여도 원금은 보호됨. 심리적 부담만 증가.
5. **Stage별 독립 실행**: A, B, C는 각각 다른 변수만 변경하므로 결과가 섞이지 않아야 함.
6. **테스트넷 정책 유지**: BYBIT_TESTNET=true 불변.
7. **결과에 따라 CLAUDE.md, orchestrator.yaml 업데이트 필요**: 최적 파라미터 확정 후.
