## 1. 결과 종합 진단

### 확정된 사실 3가지

```
[확정 1] 펀딩비 차익거래는 Phase 4 진입 가능
  - 3년 Walk-Forward Sharpe 2.434
  - 10개 윈도우 중 8개 양수 (consistency 0.80)
  - MDD 0.38% (극히 안전)
  → 이 전략은 더 이상 의심할 필요 없다. 작동한다.

[확정 2] 그리드 트레이딩은 완전 제거
  - 3년 Walk-Forward Sharpe -0.581, consistency 0.000
  - 대체 전략 3가지 모두 음수 Sharpe
  - 포트폴리오에서 "그리드 슬롯" 자체를 삭제한다
  → 더 이상 테스트하지 않는다. 끝났다.

[확정 3] DCA는 graduated 방식으로 업그레이드
  - baseline -20.43% → graduated +13.89% (+34.3%p 개선)
  - MDD 59.26% → 38.50% (20%p 축소)
  → 다만 Walk-Forward 재검증이 필요하다.
```

### 미해결 이슈 3가지

```
[이슈 1] Test B에서 4가지 변형 중 2가지만 테스트됨
  - baseline과 tolerant_exit만 실행되고 동일한 결과
  - strict_entry (진입 강화)와 short_hold (7일 보유) 결과가 없음
  → 나머지 2가지도 테스트해야 한다

[이슈 2] graduated DCA의 Walk-Forward 미검증
  - Test C에서 graduated가 최우수지만, 단일 백테스트 결과일 뿐
  - Walk-Forward로 과적합 여부를 확인하지 않았음
  → graduated DCA로 코드 업데이트 후 Walk-Forward 재실행 필수

[이슈 3] 최종 포트폴리오 구성이 미확정
  - 그리드 제거 후 전략이 2개(FA + DCA)로 줄었음
  - 가중치 매트릭스를 2개 전략 + 현금으로 재설계해야 함
  → 새로운 가중치 매트릭스 + Walk-Forward가 필요
```

## 2. 남은 테스트 목록

### 테스트 F: 펀딩비 나머지 변형 2가지

Test B에서 누락된 strict_entry와 short_hold를 실행한다.

```
[변형 3 — strict_entry]
  진입 조건 강화:
    - 펀딩비 > 0.01% (현재 0.005%의 2배)
    - 연속 양수 5회 (현재 3회)
  기대: 거래 횟수 감소, 승률 상승, 수수료 절약
  위험: 진입 기회를 너무 많이 놓칠 수 있음

[변형 4 — short_hold]
  보유 기간 축소:
    - 최대 7일 (현재 30일)
  기대: 트렌드 전환에 빠르게 대응
  위험: 잦은 진입/청산으로 수수료 증가

실행:
  python tests/backtest/bt_funding_arb_variants.py \
    --variants strict_entry,short_hold \
    --start 2023-04-01 --end 2026-03-31

비교 기준:
  baseline Sharpe 0.3303, Return +2.636%, MDD 0.6648%
  → 이것보다 Sharpe가 높거나 MDD가 낮으면 개선
  → 둘 다 나빠지면 baseline 유지 확정
```

### 테스트 G: graduated DCA Walk-Forward 검증

graduated가 단일 백테스트에서 최우수였지만 과적합 가능성을 배제해야 한다.

```
[실행 순서]

1. adaptive-dca 코드를 graduated 방식으로 업데이트
   - services/strategies/adaptive-dca/strategy.py 수정
   - config/strategies/adaptive_dca.yaml 파라미터 조정

2. Walk-Forward 실행 (3년, train=180일, test=90일)
   python services/backtester/run_walk_forward.py \
     --start 2023-04-01 --end 2026-03-31 \
     --train-days 180 --test-days 90 \
     --strategies adaptive_dca

3. 통과 기준:
   - OOS Sharpe > 0 (DCA는 보조 전략이므로 기준 완화)
   - consistency_ratio ≥ 0.50 (10개 윈도우 중 5개 이상 양수)
   - OOS MDD < 40% (baseline 59.26%에서 개선 확인)
```

### 테스트 H: 최종 포트폴리오 Walk-Forward

그리드를 제거하고 FA + graduated DCA + 현금으로만 구성한 포트폴리오의 최종 검증.

```
[새로운 가중치 매트릭스]

           │ 펀딩 차익 │ DCA(graduated) │ 현금
──────────┼──────────┼───────────────┼──────
횡보       │ 0.50     │ 0.10          │ 0.40
상승추세   │ 0.20     │ 0.40          │ 0.40
하락추세   │ 0.10     │ 0.00          │ 0.90
고변동     │ 0.40     │ 0.00          │ 0.60

설계 근거:
- 횡보: FA가 가장 안정적 (Test E Sharpe 2.72), DCA 소량
- 상승추세: graduated DCA가 강함 (Test C +13.89%), FA 보조
- 하락추세: Cash 90% 방어, FA 10%만 유지 (Test E에서 소량 FA가 소폭 개선)
- 고변동: FA 40% + Cash 60% (Test E에서 volatile FA Sharpe 2.63)

실행:
  python services/backtester/run_walk_forward.py \
    --start 2023-04-01 --end 2026-03-31 \
    --train-days 180 --test-days 90 \
    --strategies combined_v2

통과 기준:
  - OOS Sharpe ≥ 1.0
  - consistency_ratio ≥ 0.70
  - OOS MDD < 5%
```

## 3. 실행 순서 (남은 작업 우선순위)

```
[Step 1] 테스트 F — 펀딩비 나머지 변형 (30분)

  → baseline 대비 개선 여부 확인
  → 최종 펀딩비 파라미터 확정

[Step 6] 결과 문서화 + 설정 확정
  → 모든 결과를 문서에 기록
  → orchestrator.yaml, funding_arb.yaml, adaptive_dca.yaml 최종 확정
```

## 5. Phase 4 진입 후 달라지는 것

테스트 F~H를 모두 통과하면, 포트폴리오 구성이 기존 대비 크게 달라진다.

```
[기존 설계 — 4개 전략]
펀딩비 + 그리드 + DCA + 현금
→ 복잡하지만 그리드가 발목을 잡음

[최종 설계 — 2개 전략 + 현금]
펀딩비 + graduated DCA + 현금
→ 단순하지만 검증된 조합만 남김

장점:
  - 관리할 전략이 줄어듬 → 모니터링 부담 감소
  - 그리드의 드래그(-86%)가 제거 → 포트폴리오 수익률 개선
  - 코드 복잡도 감소 → 버그 가능성 감소
  - Docker 서비스 1개 제거 (grid-trading 컨테이너)

docker-compose.yml 변경:
  - grid-trading 서비스 제거 또는 주석 처리
  - Orchestrator에서 grid 관련 코드 비활성화
```

## 7. 핵심 교훈

이번 백테스트 과정에서 확인된 가장 중요한 교훈 3가지입니다.

```
[교훈 1] 전략 수가 많다고 좋은 것이 아니다

처음 설계: 4개 전략 (FA + Grid + DCA + 현금)
최종 결론: 2개 전략 (FA + graduated DCA + 현금)

그리드를 넣어서 "다양성"을 확보하려 했지만,
실제로는 그리드가 FA의 수익을 잠식하고 있었다.
combined Sharpe -0.152는 FA 단독 2.434보다 훨씬 나쁘다.
나쁜 전략을 추가하면 포트폴리오가 나빠진다. 당연하지만 쉽게 잊는 사실이다.


[교훈 2] 백테스트 기간이 결론을 좌우한다

6개월 백테스트: 모든 전략 Walk-Forward 실패 (하락장 편향)
3년 백테스트:  FA Sharpe 2.434 통과 (상승/하락 모두 포함)

만약 6개월 결과만 보고 "이 전략은 안 된다"고 결론을 내렸다면
FA라는 검증된 전략을 버릴 뻔했다.
백테스트 기간은 최소 1개 풀 사이클(상승→하락→횡보)을 포함해야 한다.


[교훈 3] 단순한 시스템이 복잡한 시스템보다 강하다

4개 전략 + 4개 레짐 + 16셀 가중치 매트릭스 → 과적합 위험 높음
2개 전략 + 4개 레짐 + 8셀 가중치 매트릭스 → 더 강건함

가중치 민감도(Test E)에서 Sharpe std 0.008이 나온 것은
시스템이 파라미터 변화에 둔감하다는 뜻이고,
이것이야말로 실전에서 오래 생존하는 시스템의 특징이다.
```
