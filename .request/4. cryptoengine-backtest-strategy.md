# CryptoEngine 백테스트 전략

> **범위**: 차트 데이터 중심 (LLM 리포트는 별도 Phase에서 추가)
> **목표**: 레짐 판별 타임프레임, 레짐 판별 로직, 가중치 매트릭스 — 이 3가지의 최적 조합을 찾는다

---

## 왜 백테스트가 복잡한가

수익에 영향을 주는 변수가 3겹으로 중첩되어 있기 때문입니다.

```
[변수 1] 레짐을 어떤 타임프레임으로 판별하는가?
         → 15분봉? 1시간봉? 4시간봉? 복합?

[변수 2] 레짐을 어떤 로직으로 판별하는가?
         → ADX 임계값? BB 폭? ATR 배수? 조합 방식?

[변수 3] 판별된 레짐에서 자본을 어떻게 배분하는가?
         → 가중치 매트릭스의 각 셀 값
```

이 3개를 동시에 최적화하면 과적합에 빠집니다. 그래서 **안쪽부터 바깥쪽으로** 한 겹씩 확정해 나갑니다.

---

## 백테스트 4단계 구조

```
┌────────────────────────────────────────────────────────┐
│ Stage 1: 개별 전략 백테스트 (가중치 무관)              │
│   → 각 전략이 단독으로 얼마나 수익을 내는지 확인      │
├────────────────────────────────────────────────────────┤
│ Stage 2: 레짐 판별 정확도 테스트                      │
│   → 15분/1시간/4시간 봉 중 어떤 것이 레짐을 정확히   │
│     분류하는지 확인                                    │
├────────────────────────────────────────────────────────┤
│ Stage 3: 가중치 매트릭스 최적화                       │
│   → Stage 2에서 확정된 레짐 판별 위에서               │
│     최적 자본 배분 비율을 찾는다                       │
├────────────────────────────────────────────────────────┤
│ Stage 4: Walk-Forward 검증                            │
│   → Stage 1~3의 결과가 미래에도 유효한지 확인         │
└────────────────────────────────────────────────────────┘
```

## Stage 1: 개별 전략 백테스트

목적은 **각 전략이 "어떤 시장 상황에서 돈을 벌고, 어떤 상황에서 잃는지"** 파악하는 것입니다. 가중치나 레짐과 무관하게 전략 하나씩 돌립니다.

### 1-1. 펀딩비 전략 단독 백테스트

```
입력 데이터: 6개월 BTCUSDT 펀딩비 히스토리 + OHLCV
자본금: $10,000 (고정)
실행: 모든 구간에서 전략 조건 충족 시 진입/청산

측정 지표:
  - 총 수익 / 연환산 수익률
  - Sharpe Ratio
  - Max Drawdown
  - 진입 횟수 / 평균 보유 기간
  - 펀딩비 양수 구간에서의 수익 vs 음수 구간에서의 손실
  - 월별 수익 분포 (어떤 달에 많이 벌고, 어떤 달에 적었는가)
```

### 1-2. 그리드 전략 단독 백테스트

```
입력 데이터: 6개월 BTCUSDT OHLCV (1분봉 이상)
자본금: $10,000 (고정)
실행: 모든 구간에서 그리드 가동

측정 지표:
  - 위와 동일 +
  - 횡보 구간 수익 vs 추세 구간 손실
  - 그리드 이탈 횟수 (범위 벗어난 횟수)
  - 최적 그리드 간격 (0.2%, 0.3%, 0.5% 비교)
```

### 1-3. DCA 전략 단독 백테스트

```
입력 데이터: 6개월 BTCUSDT OHLCV + Fear & Greed Index 히스토리
자본금: $10,000 (고정)
실행: 매주 월요일 F&G 기반 매수

측정 지표:
  - 누적 BTC 수량
  - 평균 매입 단가 vs 단순 DCA 대비 개선율
  - 포트폴리오 가치 변화
```

### Stage 1 결과로 알 수 있는 것

```
[예상 결과 패턴]

펀딩비: 상승장/횡보장에서 꾸준한 수익, 하락장에서 약한 수익 또는 0
그리드:  횡보장에서 높은 수익, 추세장에서 손실
DCA:    하락장에서 좋은 매수 기회, 상승장에서 자산 가치 상승

→ 이 결과가 "왜 레짐별로 가중치를 다르게 해야 하는지"의 근거가 됩니다.
→ Stage 3에서 가중치를 잡을 때 이 데이터를 참고합니다.
```

## Stage 2: 레짐 판별 정확도 테스트

가장 중요한 단계입니다. **레짐을 틀리게 판별하면 가중치 매트릭스가 아무리 좋아도 소용없습니다.**

### 2-1. 먼저 "정답 레짐"을 만든다

과거 데이터를 사람이 보고 구간별 레짐을 라벨링합니다. 완전히 수동으로 할 필요 없이 **후행 지표로 확정**합니다.

```python
# 후행 확정 레짐 (look-ahead 허용 — 백테스트 정답지 역할)
# 이것은 "미래를 본 후 확정한 정답"이므로 라이브에서는 사용 불가
# 오직 레짐 판별 로직의 정확도를 측정하기 위한 용도

def label_true_regime(df, window=48):
    """48캔들(4시간봉 기준 8일) 후의 결과로 현재 레짐을 확정"""
    results = []
    for i in range(len(df) - window):
        future = df.iloc[i:i+window]
        price_change = (future['close'].iloc[-1] - future['close'].iloc[0]) / future['close'].iloc[0]
        volatility = future['close'].pct_change().std() * (252**0.5)  # 연율 변동성
        
        if volatility > 0.8:  # 연 80% 이상 변동성
            regime = "volatile"
        elif price_change > 0.05:  # 5% 이상 상승
            regime = "trending_up"
        elif price_change < -0.05:  # 5% 이상 하락
            regime = "trending_down"
        else:
            regime = "ranging"
        
        results.append(regime)
    
    return results
```

### 2-2. 타임프레임별 레짐 판별 정확도 비교

동일한 판별 로직(ADX + BB + ATR)을 3개 타임프레임에서 실행하고 정답지와 비교합니다.

```

───────────┼─────────────────┼─────────────────
15분봉      │ ADX + BB + ATR  │ vs 정답 레짐
1시간봉     │ ADX + BB + ATR  │ vs 정답 레짐
4시간봉     │ ADX + BB + ATR  │ vs 정답 레짐
```

```python
# 각 타임프레임에서 레짐 판별 실행
```

### 2-3. 멀티 타임프레임 합의 (Multi-TF Consensus)

단일 타임프레임보다 **여러 타임프레임의 합의**가 더 정확할 수 있습니다.

```
[멀티 타임프레임 합의 방식]

방법 A — 다수결:
  15분봉: "ranging"
  1시간봉: "ranging"  
  4시간봉: "trending_up"
  → 2:1로 "ranging" 확정

방법 B — 가중 투표:
  15분봉 × 0.2 = "ranging"  (가중치 낮음: 노이즈 많음)
  1시간봉 × 0.3 = "ranging"
  4시간봉 × 0.5 = "trending_up"  (가중치 높음: 신호 안정적)
  → 가중 점수로 판별

방법 C — 계층적 필터:
  4시간봉으로 대분류 (trending vs ranging)
  1시간봉으로 세분류 (trending_up vs trending_down)
  15분봉은 진입 타이밍에만 사용

방법 D — 컨센서스 필터:
  3개 타임프레임이 모두 같은 레짐 → 해당 레짐 확정 (신뢰도 높음)
  2개만 일치 → 해당 레짐이지만 가중치를 보수적으로 적용
  3개 모두 불일치 → "uncertain" → 현금 비중 극대화
```

```
[테스트 매트릭스 확장]

방법                      │ 정확도   │ 레짐 전환 빈도  │ 비고
─────────────────────────┼─────────┼────────────────┼───────────────
15분봉 단독               │ 측정    │ 측정           │ 노이즈 많을 수 있음
1시간봉 단독              │ 측정    │ 측정           │ 균형점 예상
4시간봉 단독              │ 측정    │ 측정           │ 느리지만 안정적
다수결                    │ 측정    │ 측정           │
가중 투표 (0.2/0.3/0.5)  │ 측정    │ 측정           │
계층적 필터               │ 측정    │ 측정           │
컨센서스 필터             │ 측정    │ 측정           │ 가장 보수적
```

### 2-4. 레짐 전환 빈도가 왜 중요한가

레짐이 바뀔 때마다 전략이 포지션을 조정합니다. 이때 수수료가 발생합니다.

```
예시:
  1시간 안에 레짐이 3번 바뀌면 → 포지션 조정 3번 → 수수료 × 3
  이게 반복되면 "수수료로 수익이 다 날아감"

이상적인 레짐 판별:
  - 정확도 높음 (실제 시장 상태와 일치)
  - 전환 빈도 낮음 (불필요한 포지션 조정 최소화)
  - 전환 지연 짧음 (레짐 변화에 너무 늦게 반응하지 않음)

→ 이 3가지를 동시에 만족하는 타임프레임/방법을 찾는 것이 Stage 2의 핵심
```

### 2-5. Stage 2 최종 평가 지표

```python
def evaluate_regime_method(predicted, actual, transitions):
    accuracy = compute_accuracy(predicted, actual)
    transition_rate = transitions / len(predicted)  # 전환 빈도
    
    # 가짜 전환 비율: 레짐이 바뀌었다가 2캔들 내에 원래로 돌아오는 비율
    false_transition_rate = count_false_transitions(predicted) / max(transitions, 1)
    
    # 종합 점수: 정확도 높고, 전환 빈도 낮고, 가짜 전환 적을수록 좋음
    score = accuracy * 0.5 + (1 - transition_rate) * 0.3 + (1 - false_transition_rate) * 0.2
    
    return {
        "accuracy": accuracy,
        "transition_rate": transition_rate,
        "false_transition_rate": false_transition_rate,
        "composite_score": score
    }
```

## Stage 3: 가중치 매트릭스 최적화

Stage 2에서 최적 레짐 판별 방법이 확정된 후 진행합니다.

### 3-1. 현재 가중치 매트릭스 (베이스라인)

```
레짐          │ 펀딩 차익 │ 그리드 │ DCA  │ 현금
─────────────┼──────────┼───────┼─────┼─────
횡보          │ 0.25     │ 0.40  │ 0.15│ 0.20
상승 추세     │ 0.15     │ 0.05  │ 0.50│ 0.30
하락 추세     │ 0.20     │ 0.05  │ 0.10│ 0.65
고변동        │ 0.10     │ 0.05  │ 0.05│ 0.80
```

### 3-2. 가중치 탐색 방법

전체 가중치 공간을 그리드 서치합니다. 단, 과적합을 방지하기 위해 **거친 간격(0.05 단위)**으로만 탐색합니다.

```
[탐색 제약 조건]

1. 각 행의 합 = 1.0
2. 모든 값 ≥ 0.0
3. 현금 ≥ 0.10 (어떤 레짐에서도 최소 10% 현금 보유)
4. 펀딩 차익 ≤ 0.50 (단일 전략 과집중 방지)
5. 그리드 ≤ 0.50
6. DCA ≤ 0.50
7. 0.05 단위로만 탐색 (과적합 방지)
```

```python
# 가중치 그리드 서치 (개념 코드)
import itertools

def generate_weight_candidates():
    """0.05 단위로 합이 1.0인 모든 4-tuple 생성"""
    steps = [round(x * 0.05, 2) for x in range(0, 21)]  # 0.0 ~ 1.0
    candidates = []
    for f, g, d, c in itertools.product(steps, steps, steps, steps):
        if abs(f + g + d + c - 1.0) < 0.001:
            if c >= 0.10 and f <= 0.50 and g <= 0.50 and d <= 0.50:
                candidates.append({"funding_arb": f, "grid": g, "dca": d, "cash": c})
    return candidates

# 각 레짐별로 독립 탐색
for regime in ["ranging", "trending_up", "trending_down", "volatile"]:
    best_score = -inf
    for weights in generate_weight_candidates():
        result = backtest_regime_segment(regime, weights)
        score = result.sharpe  # 또는 복합 점수
        if score > best_score:
            best_score = score
            best_weights[regime] = weights
```

### 3-3. 최적화 목표 함수

단순 수익률이 아닌 **위험 조정 수익**을 최적화합니다.

```python
def objective(backtest_result):
    """최적화 목표: 안정적 수익을 우선"""
    sharpe = backtest_result.sharpe_ratio
    max_dd = backtest_result.max_drawdown
    monthly_consistency = backtest_result.pct_positive_months  # 양수 월 비율
    
    # 패널티
    if max_dd > 0.05:  # 5% 이상 낙폭이면 강력 패널티
        penalty = (max_dd - 0.05) * 10
    else:
        penalty = 0
    
    score = (sharpe * 0.4) + (monthly_consistency * 0.4) - (penalty * 0.2)
    return score
```

## Stage 4: Walk-Forward 검증

Stage 1~3에서 찾은 최적 설정이 **과거에만 잘 되는 것(과적합)**이 아닌지 확인합니다.

### 4-1. Walk-Forward 방식

```
[6개월 데이터를 4개 윈도우로 분할]

Window 1:
  학습: 2025-10 ~ 2026-01 (3개월)
  검증: 2026-01 ~ 2026-02 (1개월)

Window 2:
  학습: 2025-11 ~ 2026-02 (3개월)
  검증: 2026-02 ~ 2026-03 (1개월)

Window 3:
  학습: 2025-12 ~ 2026-03 (3개월)
  검증: 2026-03 ~ 2026-04 (1개월)

각 윈도우에서:
1. 학습 구간으로 최적 파라미터 탐색
2. 검증 구간에서 그 파라미터로 백테스트
3. 검증 구간 성과 기록

최종 판단:
  검증 구간 평균 Sharpe ≥ 1.5 → 전략 유효
  검증 구간 평균 Sharpe < 1.0 → 과적합 의심 → 파라미터 단순화
```

### 4-2. 파라미터 안정성 테스트

최적 파라미터를 ±10% 변경했을 때 성과가 급변하면 과적합입니다.

```python
        test_params = copy.copy(base_params)
        test_params[param_name] = base_value * (1 + delta)
        result = run_backtest(test_params)
        results.append(result.sharpe)
    
    # Sharpe의 표준편차가 0.3 이상이면 해당 파라미터에 과민 → 위험
    sensitivity = std(results)
    print(f"{param_name}: sensitivity = {sensitivity}")
```

## 과적합 경고 신호

아래 중 하나라도 해당하면 파라미터를 단순화해야 합니다.

```
⚠️ 학습 구간 Sharpe > 3.0인데 검증 구간 Sharpe < 1.0
   → 전형적인 과적합

⚠️ 최적 파라미터를 10% 바꿨더니 Sharpe가 절반으로 하락
   → 파라미터에 과민 반응

⚠️ 가중치 매트릭스의 어떤 셀이 0.0 또는 1.0에 가까움
   → 극단적 배분은 과적합의 신호

⚠️ 최적 가중치가 "상식"과 크게 다름
   예: 하락장에서 그리드 비중이 50% → 말이 안 됨

⚠️ 전환 빈도가 하루 10회 이상
   → 수수료로 수익이 소멸
```

## 한 가지 핵심 원칙

**백테스트에서 "가장 수익이 높은 설정"을 고르지 않습니다. "가장 안정적인 설정"을 고릅니다.**

```
설정 A: 연 35% 수익, Max Drawdown 12%, 양수월비율 70%
설정 B: 연 18% 수익, Max Drawdown 3%, 양수월비율 95%

→ 설정 B를 선택합니다.

대시보드에 12개월 연속 우상향 곡선을 그리는 것이
한 달 35% 벌고 다음 달 -12% 찍는 것보다
사람들의 신뢰를 만드는 데 100배 더 효과적입니다.
```
