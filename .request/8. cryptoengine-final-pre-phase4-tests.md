## 2. 추가 테스트 목록

### 테스트 K: 레짐 판별 안정화 (가짜 전환 감소)

가짜 전환이 99%라는 것은 레짐 판별에 확인 지연(confirmation delay)이 필요하다는 의미입니다.

```
[현재] 매 1시간봉마다 레짐 재판별 → 즉시 적용
[개선안] 레짐이 바뀌면 N봉 연속 유지되어야 확정

변형 1 — 확인 지연 2봉 (2시간):
  레짐 변경 감지 → 2봉 연속 동일 레짐 유지 → 확정
  예: 횡보 → 상승추세 1봉 → 횡보 복귀 → 전환 무시 (가짜 제거)

변형 2 — 확인 지연 4봉 (4시간):
  더 보수적. 가짜 전환 더 많이 제거하지만 진짜 전환에도 4시간 지연.

변형 3 — 확인 지연 12봉 (12시간):
  매우 보수적. 반나절 동안 레짐이 유지되어야 전환.

변형 4 — 히스테리시스(Hysteresis):
  상승→횡보: ADX < 20 (현재 그대로)
  횡보→상승: ADX > 30 (기존 25에서 30으로 올림)
  즉, "들어가는 문턱"과 "나가는 문턱"을 다르게 설정.
  문 밖에 나갔다가 바로 돌아오는 현상을 방지.

측정 지표:
  - 가짜 전환 비율 (99%에서 얼마나 줄어드는가)
  - 진짜 전환 감지 지연 (진짜 전환을 얼마나 늦게 잡는가)
  - FA OOS Sharpe (개선되는가)
  - 레짐별 체류 시간 분포 (안정적인가)

실행:
  python tests/backtest/bt_regime_stability.py \
    --delays 0,2,4,12 \
    --hysteresis 25:20,30:20,30:15 \
    --start 2020-04-01 --end 2026-03-31
```

### 테스트 M: FA 진입 파라미터 최적화 (6년 기준)

Test F에서 short_hold가 확정되었지만, 진입 조건(min_rate, consecutive)은 3년 기준으로 설정된 것입니다. 6년 데이터에서 재최적화합니다.

```
[현재 파라미터]
  min_funding_rate: 0.0001 (0.01%)
  consecutive_intervals: 3 (연속 3회 양수)
  max_holding_bars: 168 (7일)

[탐색 범위]
  min_funding_rate: [0.00005, 0.0001, 0.00015, 0.0002, 0.0003]
  consecutive: [2, 3, 4, 5]
  max_holding_bars: [72, 120, 168, 240]  (3일, 5일, 7일, 10일)

총 조합: 5 × 4 × 4 = 80가지

측정: 6년 Walk-Forward (22개 윈도우)
목표: OOS Sharpe > 1.1 (현재 기준)이면서 거래 수가 적당한(50~300회) 조합

주의: 80가지 중 "최고 Sharpe"를 뽑으면 과적합이다.
  → 상위 10개의 Sharpe 분포가 좁으면(std < 0.1) 해당 영역이 안정적
  → 상위 10개의 평균 파라미터를 최종 선택 (peak이 아닌 plateau 찾기)

실행:
  python tests/backtest/bt_fa_param_sweep.py \
    --start 2020-04-01 --end 2026-03-31 \
    --walk-forward --train-days 180 --test-days 90
```

## 4. 테스트 K 결과에 따른 분기

```
테스트 K 실행
  │
  ├─ 확인 지연 2봉 → 가짜 전환 80% 이하로 감소 + Sharpe 유지
  │   └─ 채택. orchestrator.yaml에 confirmation_bars: 2 추가
  │
  ├─ 히스테리시스 → 가짜 전환 70% 이하 + Sharpe 개선
  │   └─ 채택. regime_detector.py에 히스테리시스 로직 추가
  │
  ├─ 모든 변형에서 가짜 전환은 줄었지만 Sharpe도 하락
  │   └─ Method B가 이미 가짜 전환 비용을 0으로 만들었으므로
  │      가짜 전환 자체는 해가 안 됨. 현행 유지.
  │
  └─ 모든 변형에서 가짜 전환 감소 + Sharpe 개선
       └─ 가장 높은 Sharpe 변형 채택
```

이 분기에서 중요한 점은, **Method B 덕분에 가짜 전환의 직접 비용은 이미 0**이라는 것입니다. 가짜 전환이 해로운 유일한 경우는 "FA가 자연 청산 후 재진입할 때 잘못된 레짐의 가중치를 적용받는 것"인데, FA의 자연 청산 빈도가 7일에 1회 정도이므로 실제 영향은 제한적입니다. 따라서 테스트 K 결과가 미미해도 Phase 4 진입을 막지는 않습니다.
