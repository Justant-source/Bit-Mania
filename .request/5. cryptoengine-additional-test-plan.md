# 백테스트 결과 심층 분석 + 추가 전략 테스트 계획

> **기반 데이터**: cryptoengine-backtest-result.md (2026-04-05)
> **목적**: 결과에서 드러난 문제를 진단하고, 해결할 추가 테스트를 설계한다

---

## 1. 결과 진단 — 무엇이 문제인가

### 진단 1: 그리드 트레이딩은 죽었다

```
3년 수익률:   -86.25%
6개월 수익률: -19.49%
모든 레짐 최적 비중: 0%
```

이것은 파라미터 문제가 아닙니다. 2023~2026 BTC 시장이 **강한 추세장**(2023 상승 → 2024 상승 → 2025 하락)이었기 때문에, 횡보장에서만 수익을 내는 그리드가 구조적으로 작동할 수 없었습니다. 그리드가 잘 되려면 BTC가 수개월 동안 좁은 범위에서 움직여야 하는데, 3년간 그런 구간이 거의 없었습니다.

**결정**: 그리드를 제거하거나 완전히 다른 전략으로 교체해야 합니다. 파라미터 조절로 해결될 문제가 아닙니다.

### 진단 2: DCA는 하락장 필터가 없다

```
2023년: +23.56%
2024년: +23.96%
6개월 (하락장): -43.24%
```

현재 DCA는 F&G 지수만 보고 매수합니다. 하락장에서 F&G가 "극단적 공포"로 떨어지면 3배 매수를 하는데, 하락이 더 지속되면 손실이 눈덩이처럼 커집니다. "공포 구간에서 사면 나중에 오른다"는 논리는 **3~5년 장기**에서는 맞지만, **6개월 단위**에서는 아닐 수 있습니다.

**결정**: DCA에 추세 필터를 추가해야 합니다. 하락 추세가 확인되면 매수를 중단하거나 축소하는 로직이 필요합니다.

### 진단 3: 펀딩비 전략은 진짜 좋지만, 최근 6개월이 약하다

```
3년 Sharpe:    2.66~5.49 (탁월)
3년 MDD:       0.60% (극히 낮음)
6개월 수익률:  +0.48% (미미)
Q1 2026:       -0.17% (소폭 손실)
```

3년 전체로 보면 최고의 전략이 맞습니다. 문제는 2025-10 ~ 2026-03 구간이 하락장이라 펀딩비가 음수로 전환된 구간이 있었고, 그때 청산 비용이 발생한 것입니다. 하지만 MDD 0.60%는 놀라울 정도로 낮으므로 전략 자체는 건전합니다.

**결정**: 펀딩비 전략은 유지하되, "음수 펀딩비 구간에서의 행동"을 더 정교하게 테스트해야 합니다.

### 진단 4: Walk-Forward 실패는 시장 환경 탓이 크다

6개월 테스트 기간(2025-10 ~ 2026-03)이 하락장이었기 때문에 모든 전략이 Walk-Forward를 통과하지 못했습니다. 이것은 **"전략이 나쁘다"가 아니라 "검증 기간이 편향되어 있다"**는 뜻입니다.

**결정**: 3년 데이터로 Walk-Forward를 재실행해야 합니다. 상승/횡보/하락을 모두 포함하는 검증이 필요합니다.

## 2. 추가 테스트 계획

### 테스트 A: 3년 Walk-Forward 재실행

```
[가장 먼저 해야 할 테스트]

목적: 6개월 편향을 제거하고 공정한 검증을 한다

설정:
  데이터: 2023-04-01 ~ 2026-03-31 (3년)
  학습기간: 180일 (6개월)
  검증기간: 90일 (3개월)
  
윈도우:
  W0: 학습 2023-04~09 → 검증 2023-10~12  (상승장)
  W1: 학습 2023-07~12 → 검증 2024-01~03  (상승장)
  W2: 학습 2023-10~2024-03 → 검증 2024-04~06
  W3: 학습 2024-01~06 → 검증 2024-07~09
  W4: 학습 2024-04~09 → 검증 2024-10~12  (상승장 끝)
  W5: 학습 2024-07~12 → 검증 2025-01~03
  W6: 학습 2024-10~2025-03 → 검증 2025-04~06
  W7: 학습 2025-01~06 → 검증 2025-07~09
  W8: 학습 2025-04~09 → 검증 2025-10~12  (하락 시작)
  W9: 학습 2025-07~12 → 검증 2026-01~03  (하락 심화)

통과 기준:
  - 10개 윈도우 중 7개 이상에서 검증 Sharpe > 0
  - 평균 검증 Sharpe ≥ 1.0 (6개월 기준 1.5에서 완화)
  - 검증 구간 MDD < 5%

실행 명령:
  python services/backtester/run_walk_forward.py \
    --start 2023-04-01 --end 2026-03-31 \
    --train-days 180 --test-days 90
```

### 테스트 B: 펀딩비 전략 변형 비교

```
[펀딩비 전략을 3가지 변형으로 비교]

변형 1 — 현재 (baseline):
  음수 펀딩비 → 즉시 청산 → 양수 복귀까지 대기
  
변형 2 — 관대한 청산:
  음수 펀딩비 → 24시간 관찰 → 24시간 연속 음수일 때만 청산
  이유: 일시적 음수(몇 시간)에 매번 청산하면 수수료 낭비

변형 3 — 진입 기준 강화:
  펀딩비 > 0.01% (현재 0.005%에서 2배) 일 때만 진입
  연속 양수 3회 → 5회로 강화
  이유: 선택적으로 "확실한 구간"에서만 진입하면 승률이 올라감

변형 4 — 기간 제한:
  최대 보유 기간 7일 (현재 30일에서 축소)
  이유: 짧게 잡고 자주 재진입하면 트렌드 변화에 빠르게 대응

[4가지 변형 × 3년 데이터 = 12회 백테스트]

비교 기준:
  - 연 수익률
  - Sharpe Ratio
  - MDD
  - 거래 횟수 (적을수록 수수료 절약)
  - "음수 펀딩비 구간"에서의 손실 크기
```

### 테스트 C: DCA 추세 필터 추가

```
[현재 DCA의 문제]
F&G 지수 0~10 → 3배 매수
하지만 하락 추세에서 F&G가 계속 0~10이면 → 계속 3배 매수 → 큰 손실

[DCA 변형 비교]

변형 1 — 현재 (baseline):
  F&G 기반 매수만

변형 2 — EMA 필터:
  BTC 가격이 50일 EMA 위에 있을 때만 매수
  아래면 매수 중단 (현금 보유)

변형 3 — 이중 필터:
  F&G < 25 AND BTC가 200일 EMA 위 → 매수 (진짜 "공포 속 기회")
  F&G < 25 AND BTC가 200일 EMA 아래 → 매수 중단 (하락 추세 내 공포)

변형 4 — 단계적 축소:
  BTC가 50일 EMA 위 → 원래 배수 (1x~3x)
  BTC가 50일 EMA 아래이고 200일 EMA 위 → 배수 절반 (0.5x~1.5x)
  BTC가 200일 EMA 아래 → 매수 중단

[4가지 변형 × 3년 데이터 = 12회 백테스트]

핵심 비교 지표:
  - 3년 총 수익률
  - 하락 구간(2025-10~2026-03) 손실 크기
  - 상승 구간(2023~2024) 수익이 줄어드는 정도
  → 상승장 수익 감소 < 하락장 손실 감소면 변형이 더 나은 것
```

### 테스트 D: 그리드 대체 전략 탐색

그리드를 살리는 테스트가 아닙니다. **그리드 슬롯을 더 나은 전략으로 교체**하는 테스트입니다.

```
[후보 전략 3가지]

후보 1 — 볼린저 밴드 평균회귀 (Mean Reversion):
  BB 하단 터치 + RSI < 30 → 매수
  BB 중심(20일 MA) 복귀 → 매도
  손절: BB 하단 -1%
  
  그리드와의 차이:
  - 그리드는 "범위 내 모든 가격에 주문"이므로 추세에 취약
  - 평균회귀는 "과매도일 때만 진입"이므로 더 선택적
  - 횡보장에서 비슷한 수익, 추세장에서 손실이 작을 것으로 기대

후보 2 — Basis 스프레드 트레이딩:
  현물-선물 가격 괴리(basis)가 비정상적으로 벌어지면 진입
  평균으로 수렴하면 청산
  
  펀딩비 전략과의 차이:
  - 펀딩비는 "8시간마다 지급되는 수수료"를 수취
  - Basis는 "현물-선물 가격 차이"의 수렴을 수취
  - 두 개는 다른 수익원이므로 동시 운용 가능

후보 3 — 변동성 매도 (Volatility Selling):
  변동성이 높을 때 → 가격이 결국 평균으로 돌아온다에 베팅
  ATR이 평균의 2배 이상 → 레인지 경계에서 반대 포지션
  
  주의: 이것은 리스크가 높으므로 소량으로만 테스트

[3가지 후보 × 3년 데이터 = 9회 백테스트]

비교 기준 (그리드 대비):
  - 전체 기간 수익률 > -86.25% (그리드의 3년 성적)
  - Sharpe > -0.58 (그리드의 Sharpe)
  - 추세 구간에서의 손실 크기
  - 횡보 구간에서의 수익 (그리드의 원래 목적)
  → 사실상 "그리드보다 낫기만 하면 됨"
```

### 테스트 E: 레짐별 가중치 민감도 테스트

```
[Stage 3에서 누락된 테스트]

현재 최적 가중치를 ±10%, ±20% 변동시켰을 때 Sharpe 변화를 측정한다.

예시 (횡보 레짐, 현재 최적: FA=0.50, Cash=0.50):

  FA=0.40, Cash=0.60  → Sharpe 측정
  FA=0.45, Cash=0.55  → Sharpe 측정
  FA=0.50, Cash=0.50  → Sharpe 측정 (기준)
  FA=0.55, Cash=0.45  → Sharpe 측정
  FA=0.60, Cash=0.40  → Sharpe 측정

  Sharpe의 표준편차 < 0.3이면 → 안정적 (과적합 아님)
  Sharpe의 표준편차 ≥ 0.3이면 → 불안정 (과적합 의심)

[4개 레짐 × 5개 변형 = 20회 백테스트]
```

## 5. 테스트 결과에 따른 시나리오별 다음 단계

### 시나리오 1: 펀딩비 강하고 + 대체 전략 발견 (최선)

```
결과:
  - 3년 Walk-Forward 통과 (펀딩비 Sharpe ≥ 1.0)
  - DCA 추세 필터로 하락장 손실 -20% 이내
  - 평균회귀 또는 Basis가 Sharpe > 1.0

업데이트된 가중치 매트릭스:
  횡보:      FA 0.40 + 대체전략 0.20 + DCA 0.10 + Cash 0.30
  상승추세:  FA 0.20 + 대체전략 0.00 + DCA 0.40 + Cash 0.40
  하락추세:  FA 0.20 + 대체전략 0.00 + DCA 0.00 + Cash 0.80
  고변동:    FA 0.30 + 대체전략 0.10 + DCA 0.00 + Cash 0.60

→ Phase 4 (테스트넷) 진입
```

### 시나리오 2: 펀딩비만 유효 (현실적)

```
결과:
  - 3년 Walk-Forward 통과 (펀딩비만)
  - DCA 개선되었지만 여전히 하락장 약함
  - 대체 전략도 기대 이하

  - Phase 4 진입 보류
  - 전략 자체를 재설계
  - 펀딩비 진입 조건을 더 보수적으로 (연 환산 20% 이상만)
  - BTC 외 ETH, SOL 등 다른 자산의 펀딩비도 탐색
  - 거래소 간 펀딩비 차익(Phase 2)을 앞당겨 테스트

→ 1~2주 추가 리서치 후 재백테스트
```

### 테스트 D: 평균회귀 전략 구현 개념

```python
# services/strategies/mean-reversion/strategy.py (신규 작성 필요)

class MeanReversionStrategy(BaseStrategy):
    """
    볼린저 밴드 + RSI 기반 평균회귀.
    그리드의 "횡보장 수익" 역할을 대체하되,
    추세장에서의 손실을 최소화한다.
    """
    
    async def tick(self):
        candles = await self.get_candles("1h", 50)
        
        bb_upper, bb_middle, bb_lower = compute_bb(candles, period=20, std=2)
        rsi = compute_rsi(candles, period=14)
        adx = compute_adx(candles, period=14)
        
        current_price = candles[-1]["close"]
        
        # 추세장 필터: ADX > 30이면 거래 안 함 (그리드의 실패 원인 차단)
        if adx > 30:
            return  # 추세가 강하면 아무것도 안 함
        
        # 매수 신호: BB 하단 터치 + RSI 과매도
        if current_price <= bb_lower and rsi < 30:
            await self.submit_order(
                side="buy",
                quantity=self.calculate_position_size(),
                stop_loss=bb_lower * 0.99,    # BB 하단 -1%
                take_profit=bb_middle,          # BB 중심 (20일 MA)
            )
        
        # 매도 신호: BB 상단 터치 + RSI 과매수 (선물 숏)
        if current_price >= bb_upper and rsi > 70:
            await self.submit_order(
                side="sell",
                quantity=self.calculate_position_size(),
                stop_loss=bb_upper * 1.01,
                take_profit=bb_middle,
            )
```

모든 추가 테스트를 마친 후 확인할 것:
