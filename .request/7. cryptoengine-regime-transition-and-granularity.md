## 2. 레짐 전환 시 포지션 처리 전략

### 문제 정의

```
현재 상태: 횡보 레짐, FA 포지션 $5,000 보유 중

15분봉 기준 레짐이 "하락 추세"로 전환됨

가중치 변화:
  횡보:     FA 0.50 → $5,000
  하락추세: FA 0.10 → $1,000

질문: $5,000 포지션을 즉시 $1,000으로 줄이는가?
      아니면 기존 포지션은 유지하고 다음 거래부터 반영하는가?
```

### 3가지 옵션 비교

#### 옵션 A: 즉시 리밸런싱

```
레짐 전환 감지 → 즉시 포지션 조정

예시:
  FA $5,000 보유 중 → 하락추세 전환 → 즉시 $4,000 청산 → FA $1,000만 유지

장점:
  - 항상 현재 레짐에 맞는 포지션
  - 하락장에서 빠르게 노출 축소

단점:
  - 레짐이 자주 바뀌면 수수료 폭증
  - "가짜 전환"(30분 후 원래 레짐 복귀)에 취약
  - FA의 경우 델타 뉴트럴 포지션을 부분 청산하면
    현물/선물 수량 불일치가 발생하여 관리 복잡
```

#### 옵션 B: 신규 주문만 적용 (자연 소멸 방식)

```
레짐 전환 감지 → 기존 포지션 유지 → 자연 청산될 때까지 대기
→ 신규 진입 시에만 새 가중치 적용

예시:
  FA $5,000 보유 중 → 하락추세 전환
  → 기존 포지션은 원래 청산 조건(7일 만기, 펀딩비 역전 등)까지 유지
  → 청산 후 재진입 시 하락추세 가중치(FA 10%, $1,000) 적용

장점:
  - 수수료 절약 (불필요한 중간 청산 없음)
  - FA는 최대 7일(short_hold)이면 자연 청산되므로 지연이 길지 않음
  - 현물/선물 수량 불일치 문제 없음

단점:
  - 레짐 전환 직후 최대 7일간 과다 노출 상태 유지
  - 그 사이 급락하면 FA 단독으로는 괜찮지만 심리적 부담
```

#### 옵션 C: 단계적 전환 (하이브리드)

```
레짐 전환 감지 → 초과분의 50%만 즉시 축소 → 나머지는 자연 소멸

예시:
  FA $5,000 보유 중 → 하락추세 전환 (목표 $1,000)
  → 초과분: $5,000 - $1,000 = $4,000
  → 즉시 축소: $4,000 × 50% = $2,000 청산 → FA $3,000
  → 나머지 $2,000은 7일 내 자연 청산 시 재진입하지 않음
  → 최종적으로 FA $1,000에 수렴

장점:
  - 급격한 변화에 부분적으로 대응
  - 가짜 전환에 대한 버퍼
  - 수수료와 반응 속도의 균형

단점:
  - 로직이 복잡
  - "50%"라는 비율 자체도 최적화 대상이 되어 과적합 위험
```

### FA 전략 특성을 고려한 최종 판단

```
[핵심 사실] FA는 델타 뉴트럴이다

FA 포지션 = 현물 매수 + 선물 숏 (동일 수량)
BTC가 급락해도 FA의 손익은 거의 0이다.
따라서 "하락 추세에서 FA를 줄여야 한다"는 것은
가격 손실 방어가 아니라 "음수 펀딩비에 의한 손실 방어"이다.

음수 펀딩비 손실:
  포지션 $5,000, 펀딩비 -0.01%
  → 8시간마다 -$0.50 손실
  → 7일간 유지해도 총 -$10.50

이것은 포지션의 0.21%에 불과하다.
즉시 리밸런싱 수수료($5.50)와 비교하면 큰 차이가 없다.
```

```
[결론] FA 전략에는 옵션 B (자연 소멸)가 최적

이유:
1. FA는 델타 뉴트럴이므로 레짐 전환에 의한 가격 손실이 없음
2. short_hold(7일)이면 자연 청산까지 최대 7일 대기
3. 음수 펀딩비 손실($10.50/주)보다 리밸런싱 수수료($5.50)가 비슷하거나 더 큼
4. 가짜 전환에 면역
5. 코드 복잡도 최소

구현:
  - 레짐 전환 시 Orchestrator는 "신규 배분 자본"만 업데이트
  - 기존 FA 포지션은 원래 청산 조건(7일 만기/펀딩비 역전)까지 유지
  - 청산 후 재진입 시 새로운 레짐 가중치 적용
  - 단 하나의 예외: Kill Switch 발동 시에만 즉시 전체 청산
```

### 예외: Kill Switch는 즉시 청산

```
레짐 전환 → 자연 소멸 (수일 내)
Kill Switch → 즉시 전체 청산 (초 단위)

이 둘은 완전히 다른 이벤트이다.
레짐 전환은 "시장이 바뀌고 있다" (점진적)
Kill Switch는 "지금 당장 위험하다" (긴급)
```

### 세분화의 이론적 장단점

```
[장점]
  - 시장 상태를 더 정밀하게 포착
  - "약한 하락"과 "강한 하락"에서 다른 행동 가능
  - 2022년 폭락(강한 하락) vs 일반 조정(약한 하락) 구분

[단점]
  - 레짐 수 × 전략 수 = 파라미터 수 증가 → 과적합 위험 증가
    4개 레짐 × 3열 = 12개 파라미터
    6개 레짐 × 3열 = 18개 파라미터
    8개 레짐 × 3열 = 24개 파라미터
  - 레짐 전환 빈도 증가 → 수수료 증가
  - 각 레짐의 데이터 수가 줄어듦 → 통계적 신뢰도 하락
  - 더 복잡한 레짐 판별 로직 필요
```

### 세분화 전에 확인해야 할 것

```
[사전 확인] 각 레짐의 데이터 분포

6년(52,321봉) 중 각 레짐이 차지하는 비율을 먼저 확인:

4개 레짐 예상 분포:
  횡보:     ~40% (20,928봉)
  상승추세: ~25% (13,080봉)
  하락추세: ~20% (10,464봉)
  고변동:   ~15% (7,848봉)

8개 레짐으로 쪼개면:
  각 레짐 평균: ~12.5% (6,540봉)
  소수 레짐(극단변동, 전환구간): ~5% (2,616봉)

→ 2,616봉으로 Walk-Forward 22개 윈도우를 돌리면
  윈도우당 해당 레짐 데이터가 100봉도 안 될 수 있음
→ 통계적 의미가 약해짐

따라서 8개 레짐은 위험하고, 6개가 현실적 상한선이다.
```

```python
# 레짐 분포 사전 확인 코드
def check_regime_distribution(df, n_regimes):
    """각 레짐의 데이터 비율 확인"""
    if n_regimes == 4:
        regimes = detect_regime_4(df)
    elif n_regimes == 6:
        regimes = detect_regime_6(df)
    elif n_regimes == 8:
        regimes = detect_regime_8(df)
    
    distribution = pd.Series(regimes).value_counts(normalize=True)
    
    # 5% 미만 레짐이 있으면 경고
    thin_regimes = distribution[distribution < 0.05]
    if len(thin_regimes) > 0:
        print(f"⚠️ {n_regimes}개 레짐 중 {len(thin_regimes)}개가 5% 미만:")
        print(thin_regimes)
        print("→ 통계적 신뢰도 부족. 레짐 병합 권장.")
    
    return distribution
```

```
  → 확정된 전환 방식 + 확정된 레짐 수 + FA short_hold
  → Sharpe ≥ 1.0, consistency ≥ 0.70 확인

Step 5: Phase 4 진입
  → FA 단독 50% + 현금 50%
  → Bybit 테스트넷 4주 운영
```

## 6. Phase 4 진입 후 DCA 재추가 로드맵

DCA를 완전히 포기하는 것이 아니라, Phase 4 테스트넷 운영 중 병렬로 재설계합니다.

```
[Phase 4 기간 (4주) 중 병행 작업]

Week 1~2: FA 단독 테스트넷 운영 + DCA 재설계
  - 현재 graduated DCA의 실패 원인: EMA 후행 지표의 한계
  - 대안 설계:
    a) RSI + 볼린저 밴드 기반 과매도 진입 (EMA 제거)
    b) 펀딩비 방향과 연동 (펀딩비 양수 = 롱 심리 = DCA 활성)
    c) 단순 시간 기반 (매주 고정 금액, 필터 없음, baseline 단순화)

Week 3~4: DCA 변형 백테스트 (6년 Walk-Forward)
  - 통과 시 → Phase 4에 DCA 소량(10%) 추가
  - 실패 시 → FA + 현금 유지, DCA는 Phase 5에서 재검토

[핵심 원칙]
  DCA는 "있으면 좋지만 없어도 되는" 보조 전략이다.
  FA가 핵심이고, DCA 때문에 Phase 4를 지연시키지 않는다.
```
