# 적응형 DCA (Dollar-Cost Averaging) 전략

> [!info] 보조 전략
> **장기 축적** 목적의 보조 전략. 시장 심리 지표에 따라 매수량을 동적으로 조절합니다.
> 관련 서비스: `services/strategies/adaptive-dca/`

## 개요

시장 조건에 따라 매수 금액과 간격을 동적으로 조절하는 DCA 전략입니다. 공포 구간에서는 공격적으로 매수하고, 탐욕 구간에서는 보수적으로 운영하여 평균 매입 단가를 최적화합니다.

> [!tip] 다른 전략과의 관계
> - [[funding_arb|펀딩비 차익거래]]: 핵심 전략 — 방향성 없는 수익, DCA는 방향성(롱) 수익
> - 자본 배분은 [[architecture#2. Strategy Orchestrator|오케스트레이터]]가 레짐에 따라 관리

## 수익 구조

```
수익 = (현재가 - 평균 매입 단가) x 총 보유량 - 누적 수수료
```

- 장기 우상향 자산에서 최적
- 하락장에서 평균 단가 낮춤
- 이익 실현 래더로 일부 차익 실현

## 기본 파라미터

| 파라미터 | 기본값 | 설명 |
|----------|--------|------|
| 기본 매수 금액 | $100 | 건당 기본 금액 |
| 기본 매수 간격 | 24시간 | 기본 주기 |
| 대상 자산 | BTC, ETH | 누적 대상 |
| 포지션 방향 | 롱 전용 | 매수만 실행 |
| 레버리지 | 1배 | 레버리지 없음 |

## 적응형 멀티플라이어

### 1. Fear & Greed Index 기반

```
지수     0-10  : 극심한 공포 → 2.5배
지수    11-25  : 공포        → 1.8배
지수    26-50  : 중립        → 1.0배
지수    51-75  : 탐욕        → 0.5배
지수   76-100  : 극심한 탐욕 → 0.2배
```

- 공포가 클수록 더 많이 매수
- 탐욕이 클수록 매수량 감소

### 2. 이동평균 이탈도 기반

```
EMA200 기준:
  5% 하회  → 1.3배
  10% 하회 → 1.8배
  20% 하회 → 2.5배
  30% 하회 → 3.0배
  상회 시  → 0.7배
  20% 이상 상회 → 0.3배
```

> [!note] 기술적 지표 출처
> EMA200은 [[architecture#1. Market Data Collector|Market Data Collector]]가 계산하여 Redis에 캐싱합니다.

### 3. RSI 기반

```
RSI 14:
  < 30 (과매도) → 1.8배
  > 70 (과매수) → 0.4배
```

### 4. 변동성 (ATR) 기반

```
고변동:
  매수 간격 → 1.5배 (더 드물게)
  매수 금액 → 1.3배 (더 크게)
저변동:
  매수 간격 → 0.7배 (더 자주)
  매수 금액 → 0.8배 (더 작게)
```

### 멀티플라이어 조합

```yaml
combination_method: multiply    # 모든 배수를 곱함
min_combined_multiplier: 0.1   # 최소 0.1배
max_combined_multiplier: 5.0   # 최대 5.0배
```

**예시**: Fear 25 + RSI 28 + 가격 EMA200 대비 -12%
```
매수 금액 = $100 x 1.8(공포) x 1.8(RSI 과매도) x 1.8(이탈도)
         = $100 x 5.832
         → $500 (최대값으로 클램핑)
```

## 이익 실현 래더

| 수익률 | 매도 비율 | 설명 |
|--------|-----------|------|
| 15% | 10% | 초기 이익 확보 |
| 30% | 15% | 중간 이익 확보 |
| 50% | 20% | 상당 이익 확보 |
| 100% | 25% | 대폭 이익 확보 |

- 이익 실현 후 48시간 쿨다운 후 DCA 재개
- 남은 물량은 계속 보유

## 리스크 관리

| 항목 | 임계값 | 동작 |
|------|--------|------|
| 최대 투입 자본 | $50,000 | 추가 매수 중단 |
| 포트폴리오 비중 | 40% | 최대 배분 |
| 낙폭 중단 | -25% | 매수 일시 중지 |
| 낙폭 재개 | -15% 회복 | 매수 재개 |
| 건당 최대 | $1,000 | 멀티플라이어 적용 후 |
| 건당 최소 | $10 | 먼지 주문 방지 |
| 일일 한도 | $500 | 일일 최대 투입 |
| 주간 한도 | $2,000 | 주간 최대 투입 |

> [!warning] Kill Switch 연동
> 포트폴리오 레벨 낙폭(-5% 일일) 시 [[architecture#Kill Switch 4단계|Kill Switch L2]]가 발동되어 DCA 포지션도 청산됩니다.
> 대응 절차: [[runbook#Kill Switch 대응]]

## 운영 사이클

```
1. 매수 시점 도래 (기본 24시간 + 변동성 조정)
   ↓
2. 시장 조건 수집
   - Fear & Greed Index
   - RSI 14
   - EMA 200 이탈도
   - ATR 14 변동성
   ↓
3. 멀티플라이어 계산
   ↓
4. 매수 금액 결정 (클램핑 적용)
   ↓
5. 한도 검사 (일일/주간/총액)
   ↓
6. 주문 실행 (시장가 또는 지정가)
   → order:request 채널 경유 → 실행 엔진
   ↓
7. 이익 실현 래더 확인
   ↓
8. DB 기록 (dca_purchases 테이블)
```

> [!note] 주문 실행 흐름
> [[api#`order:request`|order:request]] → [[architecture#3. Execution Engine|실행 엔진]] → [[api#`order:result`|order:result]]

## 설정 파일

`config/strategies/adaptive_dca.yaml` 참조

## 백테스트 시 주의사항

- 장기 데이터 필요 (최소 6개월, 권장 1년 이상)
- Fear & Greed Index 히스토리 데이터 필요
- 이익 실현 시점의 슬리피지 반영
- 상승장 편향(bias) 주의 — 하락장 데이터도 포함

> [!tip] 백테스트 시스템
> [[architecture#백테스트 시스템|백테스터]]의 Monte Carlo 시뮬레이션으로 신뢰구간 확인 권장
